# CLAUDE.md

Это файл инструкций для Claude Code. Он описывает проект и рабочие процессы.

## Что это за проект

Система для создания AI-видео методом раскадровки. Вместо генерации одного длинного видео, проект разбивает процесс на управляемые шаги: бриф → референсы → кадры → фреймы → видео → склейка.

## Структура проекта

```
projects/           — рабочие проекты пользователя
templates/          — шаблоны для новых проектов
examples/           — готовые примеры
docs/               — документация
tools/              — скрипты автоматизации
final-outputs/      — финальные экспорты
```

## Правило трёх папок

Каждый этап содержит три подпапки:

```
attempts/       — черновики
approved/       — утверждённые файлы
disapproved/    — отклонённые файлы
```

Следующий этап использует ТОЛЬКО файлы из `approved/`. Это правило нельзя нарушать.

## Девять этапов рабочего процесса

1. `01-creative-brief/` — описание идеи видео
2. `02-references/` — визуальные якоря (фото продукта, персонажа, локации)
3. `03-shot-list/` — разбивка на короткие кадры
4. `04-image-prompts/` — промпты для первого и последнего фрейма каждого шота
5. `05-storyboard-frames/` — сгенерированные изображения-якоря
6. `06-video-prompts/` — промпты для генерации видеопереходов
7. `07-transition-videos/` — короткие видеоклипы между фреймами
8. `08-stitching/` — склейка клипов в единое видео
9. `09-final-output/` — финальный экспорт

## Команды для скриптов

```bash
# Создать новый проект
tools/create-project.sh название-проекта

# Настроить демо-окружение
tools/setup-demo.sh demo-walkthrough --open

# Полная настройка для новичка
tools/setup-environment.sh --open

# Сохранить на GitHub
tools/save-to-github.sh "Сообщение коммита"
```

## Режим работы

Перед началом спросить пользователя:

> Хочешь утверждать каждый этап, или мне работать автономно и останавливаться только при неясных или рискованных моментах?

Для новичков по умолчанию: утверждение на каждом этапе.

## Контрольные вопросы (approval gates)

На каждом этапе задавать соответствующий вопрос:

- Бриф: «Этот бриф описывает нужное видео?»
- Референсы: «Это те референсы, которые фиксируем?»
- Список кадров: «Правильная ли последовательность?»
- Промпты изображений: «Промпты готовы к генерации?»
- Фреймы: «Фреймы достаточно единообразны?»
- Видеопромпты: «Промпты готовы к запуску?»
- Клипы: «Клип корректно переходит между фреймами?»
- Склейка: «Это выглядит как одно непрерывное видео?»

## Важно

- НЕ генерировать видео до утверждения фреймов раскадровки
- НЕ утверждать «почти нормальные» фреймы — в видео будет хуже
- Для лиц, логотипов, продуктов — всегда использовать референс
- Короткие клипы легче контролировать, чем длинные
- Сохранять отклонённые варианты — они показывают, чего избегать
