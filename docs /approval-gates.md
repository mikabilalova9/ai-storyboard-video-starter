# Контрольные точки утверждения

Каждый этап завершается проверкой качества. Переход на следующий шаг происходит только после явного одобрения.

## Зачем нужны контрольные точки

AI-генерация накапливает ошибки. Мелкий дефект во фрейме превратится в серьёзную проблему в видео. Утверждение на каждом этапе — это фильтр, который не пропускает брак дальше по цепочке.

## Список контрольных точек

| Этап | Вопрос | Что проверять |
|------|--------|---------------|
| 1. Бриф | Бриф описывает нужное видео? | Замысел, аудитория, настроение |
| 2. Референсы | Это те референсы, которые фиксируем? | Полнота набора, качество файлов |
| 3. Кадры | Правильная ли последовательность? | Логика повествования, тайминг |
| 4. Промпты изображений | Промпты готовы к генерации? | Детализация, единообразие описаний |
| 5. Фреймы | Фреймы единообразны? | Лицо, продукт, одежда, свет |
| 6. Видеопромпты | Промпты готовы к запуску? | Движение, камера, длительность |
| 7. Клипы | Клип корректен? | Плавность перехода между якорями |
| 8. Склейка | Одно непрерывное видео? | Стыки, цвет, аудио, дрифт |

## Правила

- Файлы из `attempts/` нельзя использовать на следующем этапе
- Только файлы из `approved/` идут дальше
- Отклонённые файлы перемещаются в `disapproved/` — не удаляются
- Если ни один вариант не прошёл — генерируем новые попытки
- «Почти нормально» = не утверждено
